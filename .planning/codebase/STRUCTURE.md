## Directory Purposes

**`mods-unpacked/Pasha-AutoBattler/`:**
- Purpose: Root of the mod package, registered with Godot ModLoader
- Contains: All source code, assets, configuration, localization
- Key files: `mod_main.gd`, `manifest.json`, `autobattler_options.gd`

**`extensions/`:**
- Purpose: Godot script extension files that override base game classes
- Pattern: Mirrors game source tree structure for clarity (entities/units/player mirrors game structure)
- Pattern: `extends "res://path/to/original.gd"` as first line
- Convention: Organized by game system (singletons, entities, ui)

**`extensions/singletons/`:**
- Purpose: Global service extensions
- Contains: CoopService for multi-player bot management
- Pattern: Extends game's singleton for persistent cross-scene state

**`extensions/entities/units/`:**
- Purpose: Character and movement behavior overrides
- Contains: Player visual/config updates, movement AI logic
- Key file: `player_movement_behavior.gd` is the core steering algorithm (252 lines)

**`extensions/ui/menus/`:**
- Purpose: Menu system extensions for UI integration
- Contains: Main menu, in-game UI, global focus management, coop panel
- Pattern: Most files are placeholder extensions with minimal logic

**`ui/`:**
- Purpose: Scene and asset files specific to the mod
- Contains: AI marker scene (`AIScene.tscn`), icon texture (`defect.png`)
- Format: `.tscn` is Godot scene file (text-based), `.png` is raster image

**`translations/`:**
- Purpose: Localization assets for configuration UI
- CSV source: `autobattler_options.csv` (human-editable)
- Compiled: `autobattler_options.en.translation` (binary, generated by Godot)
- Keys: Match setting names in `manifest.json` config_schema

## Naming Conventions

**Files:**
- Snake_case for GDScript files (e.g., `player_movement_behavior.gd`, `autobattler_options.gd`)
- Lowercase with underscores for scene files (e.g., `ai_canvas.gd` → corresponds to visuals)
- Descriptive names matching game's original files (e.g., mirrors `res://` structure)

**Directories:**
- Snake_case for all directories (e.g., `mods-unpacked`, `movement_behaviors`, `extensions`)
- Feature-grouped at module level (singletons, entities, ui)
- Mirror original game structure for discoverable overrides

**GDScript Conventions:**
- Functions: snake_case (e.g., `get_movement()`, `reset_defaults()`, `setting_changed()`)
- Variables: snake_case (e.g., `enable_autobattler`, `item_weight`, `preferred_distance_squared`)
- Constants: UPPER_SNAKE_CASE (e.g., `MOD_DIR`, `DEFAULT_COOLDOWN`, `CONFIG_FILENAME`)
- Private variables: Prefixed with underscore (e.g., `_consumables_container`, `_entity_spawner`)

**Configuration Keys:**
- UPPER_SNAKE_CASE in manifest.json and autobattler_options.gd (e.g., `ENABLE_AUTOBATTLER`, `ITEM_WEIGHT`)
- Paired with human-readable labels in translations CSV (e.g., "Enable AutoBattler")
- Paired with OPTION_NAME constants for programmatic access (e.g., `ENABLE_AUTOBATTLER_OPTION_NAME`)

## Where to Add New Code

**New AI Behavior:**
- Primary code: `extensions/entities/units/movement_behaviors/player_movement_behavior.gd`
- Extend the force accumulation loop (lines 55-247)
- Pattern: Calculate `to_add` vector, validate for NaN, add to `move_vector`
- Option integration: Add weight parameter in `autobattler_options.gd`, manifest config schema

**New Configuration Option:**
- Schema definition: `manifest.json` under `config_schema.properties`
- Option singleton: `autobattler_options.gd`
  - Add variable (line 6-40 range)
  - Add OPTION_NAME constant (line 7-42 range)
  - Add case in `setting_changed()` (lines 84-115)
  - Add load/save calls (lines 130-164, 170-181)
- Localization: Add key-value pair to `translations/autobattler_options.csv`

**New Debug Visualization:**
- Primary code: `extensions/ai_canvas.gd`
- Add rendering logic to `_draw()` function (lines 3-271)
- Mirror corresponding force calculation from movement behavior
- Use `draw_circle()`, `draw_arc()` etc. for visual primitives

**New UI Integration:**

**Multiplayer/Coop Feature:**
