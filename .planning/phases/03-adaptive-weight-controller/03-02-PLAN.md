    **File 1: Create `mods-unpacked/Pasha-AutoBattlerEnhanced/extensions/adaptive_weight_controller.gd`**

    This is a standalone Node (NOT a Script Extension per D-05, ADAPT-04). It is NEVER registered in mod_main.gd.

    ```gdscript
    extends Node

    # AdaptiveWeightController — EMA-based dynamic weight adjustment.
    # Owned by AutobattlerOptions node (D-05, ADAPT-04).
    # _process() updates EMA before _physics_process() reads multipliers (D-07).

    # Group assignments (D-09): defensive weights increase under threat,
    # offensive/pickup weights decrease under threat.
    const DEFENSIVE_TYPES = ["projectile", "boss", "bumper"]
    const OFFENSIVE_TYPES = ["consumable", "item", "tree"]

    const EMA_RATE = 0.02   # D-11: ~3-second convergence at 60fps
    const MULT_MIN = 0.7    # D-10: minimum multiplier
    const MULT_MAX = 1.3    # D-10: maximum multiplier

    # EMA accumulators
    var _ema_damage_rate: float = 0.0
    var _ema_health_ratio: float = 1.0
    var _prev_health: float = -1.0    # -1.0 = sentinel for first frame (Pitfall 1)

    # Wave boundary detection (D-03)
    var _prev_wave_time_left: float = 1.0

    # Per-type multipliers (D-10: clamped [0.7, 1.3])
    var _multipliers: Dictionary = {}
    ```

    Implement these methods:

    `func _ready():` -- call `_reset()`

    `func _reset():` -- Set _ema_damage_rate=0.0, _ema_health_ratio=1.0, _prev_health=-1.0, _prev_wave_time_left=1.0. Initialize _multipliers dict with 1.0 for each key in DEFENSIVE_TYPES + OFFENSIVE_TYPES.

    `func get_multiplier(force_type: String) -> float:` -- Return `_multipliers.get(force_type, 1.0)`. The 1.0 fallback handles unknown types (egg_weight passes through unmodified).

    `func _process(delta):` -- Access player via `$"/root/Main"._players[0]` (per Research Open Question 3). Access wave timer via `$"/root/Main"._wave_timer.time_left`. Check wave boundary: if `_prev_wave_time_left < 0.05 and wave_time_left > 1.0`, call `_reset()`, set `_prev_wave_time_left = wave_time_left`, and return early. Otherwise update `_prev_wave_time_left = wave_time_left`. Then call `_update_ema(player, delta)` and `_update_multipliers()`.

    `func _update_ema(player, delta):` -- Per D-01: `var cur_hp = float(player.current_stats.health)`. If `_prev_health < 0.0` (sentinel), set `_prev_health = cur_hp` and return (skip first frame per Pitfall 1). Compute `var max_hp = float(player.max_stats.health)`. Compute `var raw_damage_rate = clamp((_prev_health - cur_hp) / (delta * max_hp), 0.0, 1.0)`. Set `_prev_health = cur_hp`. Compute `var raw_health_ratio = cur_hp / max_hp`. Apply EMA: `_ema_damage_rate = lerp(_ema_damage_rate, raw_damage_rate, EMA_RATE)` and `_ema_health_ratio = lerp(_ema_health_ratio, raw_health_ratio, EMA_RATE)`.

    `func _update_multipliers():` -- Per D-09: `var threat_level = 1.0 - _ema_health_ratio`. For each type in DEFENSIVE_TYPES: `var target = lerp(MULT_MIN, MULT_MAX, threat_level)`, then `_multipliers[t] = clamp(lerp(_multipliers[t], target, EMA_RATE), MULT_MIN, MULT_MAX)`. For each type in OFFENSIVE_TYPES: `var target = lerp(MULT_MAX, MULT_MIN, threat_level)`, then `_multipliers[t] = clamp(lerp(_multipliers[t], target, EMA_RATE), MULT_MIN, MULT_MAX)`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Player health data -> EMA | Raw health values from game engine enter the adaptive controller |
| Wave timer -> Reset trigger | Timer state from game scene triggers controller reset |
| Multiplier output -> Force weights | Controller output modifies AI behavior strength |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | Multiplier values affecting AI behavior | mitigate | Clamp all multipliers to [0.7, 1.3] in `_update_multipliers()` with `clamp()` call. No external input can push multipliers outside this range. |
| T-03-05 | Denial of Service | Division by zero in damage_rate computation | mitigate | `delta` comes from Godot engine (never zero in practice). `max_hp` guarded by clamp of damage_rate to [0.0, 1.0]. First-frame sentinel prevents uninitialized _prev_health computation. |
| T-03-06 | Elevation of Privilege | Controller running _process twice (Script Extension double-call) | mitigate | Controller is plain Node child (not Script Extension). Never registered in mod_main.gd. ADAPT-04 requirement enforces this structurally. |
| T-03-07 | Spoofing | Wave timer read from scene tree | accept | Timer access uses established mod pattern (`$"/root/Main"._wave_timer`). No untrusted code can intercept this path in single-player Brotato. |
| T-03-08 | Information Disclosure | EMA state leaks game state | accept | No PII. EMA values are derived from in-game health which is already visible to the player. No network exposure. |
</threat_model>
